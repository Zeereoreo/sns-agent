# SNS Agent — 네이버 블로그 무인 발행 시스템

메이드어스(made-us) 콘텐츠를 **자동 생성·이미지 삽입·태그·발행**하는 에이전트.
현재 **하루 3회 무인 발행**까지 동작합니다. (테스트 블로그: made-us2)

> ⚠️ 현실: 네이버는 자동 게시를 탐지합니다. 안전 페이싱(하루 3회·시간 분산)을 지키고,
> 검색의도형·정보성 글로 갑니다. 상세 리스크·전략은 `~/.claude/plans/calm-painting-platypus.md`.

## 사진 넣는 법 (당신 몫)
- **가장 쉬운 법**: 대시보드 **이미지 탭**에서 사진 선택 + 세그먼트(a 방송/b 클럽/c 간판) 골라 업로드 → 인박스로 들어가 다음 글 우선 사용.
- **폴더로**: 평소 재활용은 `drafts/photos/`, 우선 사용은 `drafts/photos/inbox/`.
- 파일명 앞에 `a_/b_/c_` 붙이면 해당 주제 글에만 쓰입니다. 사진 없어도 주제 매칭 인포그래픽으로 게시됩니다.

## 효과 측정 (에이전트가 성과를 내는지 증명)
`metrics.py` 가 매 발행 때 자동으로 지표를 쌓습니다(발행 스케줄러가 호출):
- **방문자 추이**: 블로그 홈의 오늘/전체 방문자를 매일 스냅샷 → 대시보드에 일별 막대그래프
- **키워드 검색순위**: 발행글의 타깃 검색어를 네이버 블로그탭에서 검색해 우리 글 순위를 하루 1회 기록
  → 대시보드에 키워드별 순위 + 전일 대비 화살표. "타깃 N개 중 M개가 1페이지 노출" 요약.
- 저장: `data/metrics.json` (날짜별 시계열). 수동 수집: `python metrics.py collect --ranks`, 조회: `python metrics.py show`
- **검색량(월간 검색수)**: 절대 검색량은 네이버 검색광고 API 키가 필요합니다(무료 발급). 키를 주시면
  키워드별 월간 검색수 컬럼을 추가합니다. 지금은 순위/방문자로 성과를 봅니다.

- **성과** — 방문자 추이 그래프 + 키워드 검색순위 + CSV 내보내기(클라이언트 보고용)
- **성장엔진** — 발행 우선순위·세그먼트 성과·자가 튜닝 이력·방문자 연결(알고리즘을 눈으로 평가)
- **발행** — 발행 큐(클릭 시 미리보기) + 최근 발행 기록
- **캘린더** — 미발행 초안이 언제 나갈지 날짜별 예정표
- **콘텐츠·SEO** — 전체 초안 SEO 점수(낮은 순) + 키워드별 경쟁 글 분석
- **이미지** — 웹에서 사진 업로드(세그먼트 선택→인박스) + 사진풀 썸네일
- **설정** — 주요 강조 포인트 편집(글에 자동 삽입) + 운영설정 + 테스트 알림
- **진단** — 세션/초안/이미지/작업/큐 일괄 점검
- **상태** — 자동 실행 스케줄·세션 상태·실행 화면·로그

- **끄기**: 웹페이지 오른쪽 위 **■ 종료** 버튼 (콘솔이 없어 Ctrl+C 를 못 쓰므로)
- 아이콘을 두 번 눌러도 서버가 중복 실행되지 않고 기존 탭만 열립니다
- 서버 없이 파일로만 보고 싶으면: `dashboard.py --snapshot` → `data/dashboard.html`

> 아이콘은 `.lnk` 바로가기(대상 `.venv\Scripts\pythonw.exe dashboard.py`)입니다.
> exe 로 굳이 빌드하지 않은 이유: 이 앱은 어차피 프로젝트 폴더의 초안·상태 파일을
> 읽어야 해서 폴더를 벗어나면 동작하지 않고, exe 로 묶으면 코드 고칠 때마다
> 재빌드해야 합니다. 바로가기는 항상 최신 코드를 실행합니다.
30초마다 자동 새로고침. 문제가 있으면 상단에 빨간 배너로 뜹니다(세션 만료, 배터리 차단, 사진 풀 없음, SEO 저점 대기글 등).
읽기 전용이 기본이나, 안전한 쓰기(사진 업로드·강조포인트 저장·순위 새로고침)는 지원.

## 성장 엔진 (방문자 늘리는 알고리즘)
`growth.py` 가 데이터로 **어떤 글을 먼저 발행할지**를 정하고, 결과를 보고 **스스로 가중치를 보정**한다.
- 점수 = 세그먼트 성과(발행글 순위) + SEO + 탐색 + 폭 의 가중합(전부 설명가능). 같은 세그먼트 3연속 회피.
- 발행 후 매일 자가 튜닝(성과 좋은 신호↑). 방문자 추세·순위상관도 평가(대시보드 성장엔진 탭).
- 정직: 없는 방문자를 만들지 못함. **이길 수 있는 주제에 힘을 몰아주는 최적화**. 데이터 쌓일수록 똑똑해짐.

## 실패는 조용히 넘어가지 않는다 (2026-07-20 개편)
예전엔 `naver.publish()` 가 모든 실패를 print 만 하고 None 을 반환해서,
스케줄러의 `ok = not dry_run` 이 **항상 True** 였다. 세션이 만료돼 아무것도
안 올라가도 초안이 "발행됨"으로 큐에서 영구 제거됐다.

지금은:
- `publish()` 가 `{ok, reason, images_inserted, url, title}` 를 반환한다
- `ok=True` 의 뜻은 **블로그 목록에서 그 제목을 실제로 확인했다** (버튼 클릭 성공이 아님)
- 실패하면 초안을 큐에 **남겨두고** 다음 실행에서 재시도한다
- 제목/본문 입력이 실패하면 반쪽짜리 글을 올리지 않고 중단한다
- 실패 사유(`session_expired`, `not_found_after_publish` 등)가 상태 파일에 남고
  대시보드 상단에 빨간 배너로 뜬다
