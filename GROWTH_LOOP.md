# 자율 성장 루프 (Growth Loop) — 영속 플레이북

이 에이전트가 **스스로 계속 성장**하기 위한 작업 플레이북. 사용자가 "진행해"류로
트리거하면 아래 사이클을 반복한다. (기존 스크래치패드 BACKLOG.md 를 저장소로 승격 —
compact·세션 종료에도 살아남게.)

## 목표
네이버 블로그 **방문자 성장**. 순위 1위가 아니라 실제 방문자가 늘어야 한다.

## 매 주기 사이클
1. **측정** — 읽기: `scheduler.py status`, `growth.py report`, `demand.py audit`(초안 키워드 수요),
   `data/metrics.json`(방문자·순위·세션), 이 파일의 진행 로그.
2. **진단** — 방문자 성장을 막는 **가장 큰 레버 1개** 식별.
3. **구현** — 그 레버 1개만 개선. 값 없으면 정직히 "조치 불필요 + 라이브 점검"만.
4. **검증** — `test_smoke.py` 통과. 발행 로직 변경 시 `scheduler.py run --dry-run`.
5. **평가·기록** — git 커밋·푸시. 이 파일 "진행 로그" 갱신(무엇/왜/결과).
6. **자기평가** — 다음 주기에 방문자·순위로 이어졌는지 확인.

## 불변 규칙
- 억지 작업 금지. 정직(가짜 데이터·귀속 안 만듦). 라이브 무중단(dry-run만).
- drafts/ 는 gitignore(라이브 반영, git 커밋 없음). 새 글은 seo.py A등급 확인.
- 자동생성(Anthropic 키)·인스타/X 는 사용자 지시로 보류. 네이버 집중.

## 핵심 인사이트
- 방문자 = 검색수요 있는 키워드 × 순위 × 블로그지수. 지수는 시간(수 주)이 필요.
- C(간판) 세그먼트에 검색수요 집중(카페/병원/상가/미용실 간판, 네온사인, 아크릴 간판/메뉴판).
- A(BJ 방송피켓)는 검색이 거의 없는 니치 → 방문자 기대 낮음. 억지 리타겟 금물.
- **수요≠승산(2026-07-22 실측)**: 신생 블로그는 head term 못 이긴다. 라이브 순위: 죽은키워드(비제이피켓·LED시그니처) 1위 / 리타겟한 수요키워드(아이스버킷·네온사인·간판디자인) **전부 30위 밖**. 수요 프록시(자동완성)는 검색활동만 재고 경쟁강도는 못 잰다.
- **버킷(B)=검색 승산 없음**: "아이스버킷" 중간꼬리(LED/커스텀/클럽/샴페인/제작/렌탈/가격) 전부 수요0. head "아이스버킷"(10)은 ALS챌린지 오염+경쟁. 큐가 버킷 head 7편에 쏠린 건 착시.
- **네온사인·간판(C)=중간꼬리 승산 있음**: "네온사인 제작"(3)·"상가 간판 제작"(2)·"led 간판 제작"(2)·"카페/아크릴/돌출 간판 제작"(1). **구체적 2~3어절 수요키워드가 신생 블로그의 승부처.**
